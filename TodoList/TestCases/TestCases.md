*/ 1. Пользователи и авторизация.

1.1. Создание нового пользователя.
Предусловие: программа открыта, нет профиля.
Шаги поверки:
1. Открыть программу.
2. После текстовой строки ввести букву “n”.
3. Ввести логин и пароль.
4. Ввести имя, фамилию, год рождения.
Ожидаемый результат: профиль создан, можно начать пользование программой.
Скриншот:


1.2. Авторизация пользователя.
Предусловие: программа открыта, профиль создан.
Шаги проверки:
1. Открыть программу повторно, в поле ввести букву "y".
2. Ввести корректные логин и пароль.
Ожидаемый результат: авторизация проходит успешно.
Скриншот:

1.3. Ошибка авторизации.
Предусловие: программа открыта, профиль создан.
Шаги проверки: 
1. Открыть программу повторно, в поле ввести букву "y".
2. Ввести некорректный логин или пароль.
Ожидаемый результат: программа выдаст критическую ошибку об отсутствии профиля с данным логином.
Скриншот:

1.4. Выход из текущего пользоввателя.
Предусловие: авторизация прошла успешно.
Шаги проверки:
1. Ввести команду "profile --out".
2. Закрыть программу.
Ожидаемый результат: задачи сохранены, программа завершена.
Сериншот: 

1.5 Повторная авторизация после выхода.
Предусловие: ранее был выполнен выход из созданного профиля.
Шаги проверки:
1. Перезапустить программу.
2. Выбрать опцию входа в существующий профиль.
3. Ввести корректные данные.
Ожидаемый результат: авторизация проходит успешно.
Скриншот:

1.6. Корректное отображение данных пользователя.
Предусловие: профиль создан, пользователь авторизирован.
Шаги проверки:
1. Ввести команду profile
Ожидаемый результат: программа выведет данные пользователя, которые были введены при создании профиля.

3. Просмотр задач (view).

3.1. Просмотр списка задач.
Предусловие: авторизация выполнена, добавлены некоторые задачи.
Шаги проверки:
1. В командной сроке ввести "view"
Ожидаемый результат: программа покажет все добавленные ранее задачи (названия будут выводиться не полностью).
Скриншот:

3.2. Корректное отображение при разных флагах.
Предусловие:авторизация выполнена, добавлены некоторые задачи.
Шаги проверки:
1. Для проверки флага индекса, ввести команду view -i, либо view --index.
2. Для проверки флага статутов, ввести команду view -s, либо view --statuses.
3. Для проверки флага времени и даты, ввести команду view -d, либо view --update-date
4. Для того, чтобы показать все флаги сразу, ввести команду view -a, либо view --all.
Ожидаемый результат: программа будет корректно показывать каждый из выбранных флагов. При выборе индекса будет показаны номера задач. На втором пункте будет показано на каком этапе сейчас задача (не начато, в процессе, выполнено, отложено, провалено). При выборе флага времени и даты, будует показана дата создания задачи. Если установить флаг, показывающий все сразу, программа будет выводить индекс, статус и дату одновременно. 
Скриншот:

3.3. Просмотр задач при пустом списке.
Предусловие: авторизация выполнена, задачи отсутствуют изначально, либо же удалены уже существующие.
Шаги проверки:
1. Ввести команду view при отсутствии задач.
Ожидаемый результат: программа выдаст сообщение о том, что задач нет.
Скриншот:

4. Обновление задач.

4.1. Обновление текста задачи
Предусловие: авторизация выполнена, добавлена хотя бы одна задача.
Шаги проверки: 
1. Ввести команду update *номер задачи* и ввести текст, например: "update 3 text".
Ожидаемый результат: будет выведено сообщение об обновлении задачи, ее прежний текст и тот, на который был заменен.
Скриншот:

4.2. Обновление статуса задачи.
Предусловие: авторизация выполнена, добавлена хотя бы одна задача.
Шаги проверки: 
1. Ввести команду status *номер задачи* *наименование статуса*, например: "status 1 failed". Проделать так с каждым существующим статусом (failed, notstarted, completed, postponed)
Ожидаемый результат: статус успешно изменент, при команде view -s рядом с задачей будет отображаться ее текущий статус.
Скриншот:

4.3. Попытка обновить несуществующую задачу.
Предусловие: авторизация выполнена, задача, которую нужно обновить, не существует.
Шаги проверки:
1. Ввести команду обновления задачи и написать индекс несуществующей задачи (update 67 ronaldo)
Ожидаемый результат: программа выдаст ошибку, сообщающую, что задачи с таким номером не существует.
Скриншот:

4.4. Проверка отображения обновленного статуса.
Предусловие: авторизация выполнена, статус задачи обновлен.
Шаги проверки:
1. Ввести команду read *номер задачи*, для той задачи, статус которой был обновлен.
Ожидаемый результат: программа выведет указанную задачу с тем статусом, который был выбран, вместе с ее текстом и датой изменения.
Скриншот:
